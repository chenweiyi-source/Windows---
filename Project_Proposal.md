# Project Proposal — DailyNote 日记本应用

## 1. 项目概述与目标

DailyNote 是一款基于 Windows Forms 的桌面日记本应用，旨在为用户提供简洁、美观、易用的日记记录与管理体验。用户可以通过该应用新建日记、浏览日记列表、编辑已有日记、按关键字搜索日记，所有数据以 JSON 格式持久化存储在本地。

**核心目标：**
- 运用 WinForm 控件编程与 GDI+ 自绘技术构建精致用户界面
- 实现完整的日记 CRUD（增删改查）功能
- 通过 JSON 文件实现数据持久化
- 以 AI 辅助编程方式探索 RDD（需求驱动开发）与 TDD（测试驱动开发）相结合的模式

## 2. 功能需求与用户故事

| 编号 | 功能 | 用户故事 |
|------|------|----------|
| F1 | 新建日记 | 作为用户，我想要填写标题、正文、选择日期和心情标签，以便记录当天的心情和事件 |
| F2 | 浏览日记列表 | 作为用户，我想要按日期倒序查看所有日记，每项显示日期、标题和心情，隔行交替背景色，以便快速浏览 |
| F3 | 编辑日记 | 作为用户，我点击列表中的日记项可以加载到编辑区进行修改并保存 |
| F4 | 删除日记 | 作为用户，我可以删除不再需要的日记，删除前有确认提示防止误操作 |
| F5 | 搜索日记 | 作为用户，我在搜索框输入关键字可以实时过滤日记列表，按标题或正文内容匹配，忽略大小写 |
| F6 | 数据持久化 | 作为用户，我关闭程序后再次打开时，之前保存的日记不会丢失 |

## 4. 系统架构设计

```
┌─────────────────────────────────────────────────────┐
│                    表现层 (UI)                        │
│  Form1 ── 左侧 Panel (侧边栏) ── 右侧 Panel (编辑区) │
│    ├── ListBox (日记列表, OwnerDrawVariable)          │
│    ├── TextBox (搜索框, 标题, 正文)                  │
│    ├── DateTimePicker (日期选择)                     │
│    ├── ComboBox (心情选择, 5种心情)                   │
│    ├── Label (Logo, 副标题, 状态栏)                  │
│    └── Button (写日记/保存/删除/清空)                 │
├─────────────────────────────────────────────────────┤
│                    业务逻辑层                         │
│    ├── NewEntry — 新建/清空表单                        │
│    ├── SaveEntry — 保存(新建/编辑) + 非空校验          │
│    ├── DeleteEntry — 删除 + 确认对话框                 │
│    ├── RefreshList — 搜索过滤 + 日期倒序排列           │
│    ├── LoadEntry — 选中项加载到编辑区                   │
│    └── ClearForm — 清空表单字段                        │
├─────────────────────────────────────────────────────┤
│                    数据层                             │
│    ├── DiaryEntry 数据模型 (Id/Title/Content/Mood/Date/CreatedAt)
│    ├── LoadData() — 从 diaries.json 反序列化加载       │
│    └── SaveData() — 序列化写入 diaries.json            │
└─────────────────────────────────────────────────────┘
```

**分层说明：**

- **表现层**：负责所有 UI 布局和用户交互，所有控件通过纯代码方式创建和配置（非设计器拖拽），精确控制位置、大小、颜色和字体
- **业务逻辑层**：处理日记的增删改查、搜索过滤、表单状态管理（通过 `_isEditing` 标志区分新建/编辑模式）
- **数据层**：定义 DiaryEntry 数据模型，通过 `System.Text.Json` 实现数据读写，包含完整异常处理

## 6. 模块功能说明

### 6.1 数据模型模块

定义 `DiaryEntry` 类，包含 6 个字段：
- `Id` — GUID 前 8 位，全局唯一标识，自动生成
- `Title` — 日记标题（保存时非空校验）
- `Content` — 日记正文
- `Mood` — 心情标签（😊 开心 / 😐 一般 / 😢 难过 / 🔥 兴奋 / 😴 疲惫）
- `Date` — 日记日期（默认为当天）
- `CreatedAt` — 创建时间戳

### 6.2 数据持久化模块

- 使用 `System.Text.Json` 将 `List<DiaryEntry>` 序列化为 JSON
- 存储于程序运行目录下的 `diaries.json`
- 程序启动时自动加载 (`LoadData`)，修改后自动保存 (`SaveData`)
- 设置 `WriteIndented = true` 使 JSON 格式化输出便于调试
- 完整 try-catch 异常处理，加载失败时初始化为空列表

### 6.3 日记列表展示模块

- ListBox 设置为 `OwnerDrawVariable` 模式，通过 `DrawItem` 和 `MeasureItem` 事件自定义绘制
- 每项高度 44 像素，显示日期（8pt，靠左）、标题（10pt 加粗，超长截断为 "xxx..."）、心情标签
- 偶数行和奇数行使用不同背景色（深色交替：`#3A3F55` / `#353A4E`）
- 选中行使用主题色（`#EB9150`）高亮
- 按日期倒序排列，相同日期按创建时间倒序

### 6.4 日记编辑模块

- 右侧编辑面板包含：日期选择器、心情下拉框、标题输入框、正文多行文本框
- 通过布尔标志 `_isEditing` 区分新建/编辑模式
- 新建时清空表单并自动聚焦标题输入框
- 编辑时选中日记的数据加载到各控件中
- 保存时对标题进行非空校验
- 删除操作弹出确认对话框防止误删

### 6.5 搜索模块

- 基于 `TextChanged` 事件实时触发列表刷新
- LINQ `Where` + `string.Contains` 模糊匹配（标题和正文）
- `StringComparison.OrdinalIgnoreCase` 忽略大小写
- 关键字为空或空白字符时显示全部日记
- 搜索结果保持日期倒序排列

### 6.6 UI 配色方案

- 侧边栏深色（`#32374B`），右侧编辑区暖白色（`#F8F3EB`）
- 主题色橙色（`#EB9150`）用于新建按钮和选中高亮
- 整体温暖、舒适风格，与功能定位匹配

## 7. 开发模式：RDD + TDD

本项目采用 **RDD（需求驱动开发）** 与 **TDD（测试驱动开发）** 相结合的模式：

**RDD 层面**：先明确 6 项功能需求（F1-F6），设计 UI 布局，再进行编码实现。

**TDD 层面**：坚持"测试先行"原则，在编码前先在 `Testing_Report.md` 中编写测试用例定义验收标准，再进行实现。共设计了 15 个测试用例覆盖所有核心功能和边界场景。

**AI 协作**：使用 Claude Code 辅助代码生成。开发流程为：
1. 编写测试用例 → 2. 设计 Prompt → 3. AI 生成框架代码 → 4. 人工审核修正
